PySchool - Schema do Banco de Dados (MySQL Local)

Especificação das tabelas, relações e estrutura para o backend da plataforma PySchool.

Visão Geral das Relações (ERD)

users (1) <---> (N) user_lessons_progress
lessons (1) <---> (N) user_lessons_progress
modules (1) <---> (N) lessons
tracks (1) <---> (N) modules
lessons (1) <---> (N) exercises
users (1) <---> (N) exercise_submissions
exercises (1) <---> (N) exercise_submissions
users (1) <---> (N) user_achievements
achievements (1) <---> (N) user_achievements

Script de Criação (DDL)

-- Criando o banco de dados local
CREATE DATABASE IF NOT EXISTS pyschool_db;
USE pyschool_db;

-- Tabela: tracks (Trilhas de aprendizado)
CREATE TABLE tracks (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(100) NOT NULL,
    description TEXT,
    icon VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Tabela: modules (Módulos dentro das trilhas)
CREATE TABLE modules (
    id INT AUTO_INCREMENT PRIMARY KEY,
    track_id INT NOT NULL,
    title VARCHAR(100) NOT NULL,
    order_number INT NOT NULL,
    FOREIGN KEY (track_id) REFERENCES tracks(id) ON DELETE CASCADE
);

-- Tabela: lessons (Lições teóricas e práticas)
CREATE TABLE lessons (
    id INT AUTO_INCREMENT PRIMARY KEY,
    module_id INT NOT NULL,
    title VARCHAR(150) NOT NULL,
    content_markdown LONGTEXT, -- Conteúdo da lição em MD
    xp_reward INT DEFAULT 10,
    order_number INT NOT NULL,
    FOREIGN KEY (module_id) REFERENCES modules(id) ON DELETE CASCADE
);

-- Tabela: exercises (Desafios de código)
CREATE TABLE exercises (
    id INT AUTO_INCREMENT PRIMARY KEY,
    lesson_id INT NOT NULL,
    title VARCHAR(150) NOT NULL,
    description TEXT,
    starter_code LONGTEXT, -- Código inicial exibido no editor
    test_code LONGTEXT, -- Script de testes unitários (rodado no backend)
    xp_reward INT DEFAULT 50,
    FOREIGN KEY (lesson_id) REFERENCES lessons(id) ON DELETE CASCADE
);

-- Tabela: users (Usuário local - Single user por padrão, mas escalável)
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    total_xp INT DEFAULT 0,
    current_streak INT DEFAULT 0,
    last_login DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Tabela: user_lessons_progress (Relacionamento N:N - Progresso do usuário nas lições)
CREATE TABLE user_lessons_progress (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    lesson_id INT NOT NULL,
    status ENUM('pending', 'completed') DEFAULT 'pending',
    completed_at TIMESTAMP NULL,
    UNIQUE KEY uniq_user_lesson (user_id, lesson_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (lesson_id) REFERENCES lessons(id) ON DELETE CASCADE
);

-- Tabela: exercise_submissions (Histórico de envios de código)
CREATE TABLE exercise_submissions (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    exercise_id INT NOT NULL,
    submitted_code LONGTEXT NOT NULL,
    is_correct BOOLEAN DEFAULT FALSE,
    output_log TEXT, -- Saída do console ou erro
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (exercise_id) REFERENCES exercises(id) ON DELETE CASCADE
);

-- Tabela: achievements (Selos/Conquistas)
CREATE TABLE achievements (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    icon VARCHAR(50),
    criteria VARCHAR(255) -- Ex: 'lessons_completed:10' ou 'xp:1000'
);

-- Tabela: user_achievements (Conquistas desbloqueadas pelo usuário)
CREATE TABLE user_achievements (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    achievement_id INT NOT NULL,
    awarded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uniq_user_ach (user_id, achievement_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (achievement_id) REFERENCES achievements(id) ON DELETE CASCADE
);

Notas de Implementação para o Backend

  • Execução de Código: A tabela exercise_submissions guarda o código, mas a execução real do Python acontece no filesystem do localhost. O backend deve escrever o submitted_code e o test_code em um arquivo temporário (ex: temp_test.py), rodar via subprocess.run(['python', 'temp_test.py']) e salvar o resultado em output_log e is_correct.
  • Gamificação (XP): Ao inserir um registro em user_lessons_progress com status 'completed' ou atualizar exercise_submissions.is_correct para TRUE, o backend deve disparar um trigger ou evento de serviço para somar o XP correspondente em users.total_xp.
  • Ambiente Local: O MySQL deve ser configurado com usuário root e senha em branco (ou uma senha padrão definida no arquivo .env do backend) para facilitar o setup na máquina do desenvolvedor/estudante.
